Escalada

Enumerar todo el sistema. Estamos como el usuario NU_1055

Hay varios usuarios a nivel de Dominio. Los metemos un diccionario para probar un ASREPRoast. Pero no se puede con ninguno.

En C:\Windows\SYSVOL\domain\scripts tenemos directorios con scripts .bat, hay que encontrar los que no estén vacíos.

No encontramos muchas cositas, lo mejor será enumerar con BloodHound. Generamos el comprimido de la información del dominio y nos lo traemos. Lo añadimos al BloodHound y analizamos todo.

Vamos al usuario NU_1055 (le damos doble click) -> Node Info -> Outbound Object Control (First Degree Object Control). Podemos hacer un WriteSPN sobre el usuario RSA_4810 1.- Le asignamos un SPN al usuario. 2.- Solicitamos un service ticket para dicho SPN. 3.- Subimos el mimikatz.exe para obtener un .kirbi 4.- kirbi2john para obtener un hash y crackearlo. En el apartado de "Enumerar en BloodHound" vemos cómo llevarlo a cabo. Obtenemos la password de RSA_4810 RSA_4810:(Ni7856Do9854Ki05Ng0005 #)

Nos conectamos con evil-winrm con dichas credenciales. Checamos nuevamente en BloodHound. Analysis -> Dangerous Privileges (Find Principals with DCSync Rights). Vemos varios usuarios, entre ellos el SSA_6010 (que tiene su directorio en C:\Users)

Subimos el PowerView.ps1 (usaremos sus funciones), importamos el módulo y enumeramos el usuario SSA_6010 y de todo. Get-DomainUser ssa_6810 y vemos que inicia sesión cada minuto. Hay una función interesante Find-InterestingDomainAcl : para ver propiedades que podemos alterar como el usuario actual en el que estamos. Tenemos sobre SSA_6010 ActiveDirectoryRights : WriteProperty ObjectAceType : Script-Path

Script-Path es delicado, porque podemos modificar lo que queremos que el usuario haga en el inicio de sesión. Por lo regular está en SYSVOL. Recordando los .bat que habíamos encontrado, ya podemos entenderlos mejor. C:\Windows\SYSVOL\sysvol\blazorized.htb\scripts

icacls * : vemos que en uno tenemos control total y de forma recursiva. Todo lo que cree dentro del directorio, se ejecutará cuando el usuario inicie sesión. Así que podríamos ir a revshells.com y crear un payload en PS (base64) para entablarnos una reverse shell. Para variarle, porque se puede de muchas maneras. echo 'payloadPSBase64' | Out-File -FilePath reverse.bat -Encoding ASCII : dentro del directorio con permisos de escritura.

Pero falta algo, al hacer Get-ADUSer ssa_6010 -Properties ScriptPath : vemos que el ScriptPath está vacío, porque parte de la ruta C:...\scripts, es decir, ya toma ese directorio, pero me falta que conozca el .bat que creamos, hacemos Get-ADUSer ssa_6010 | Set-ADUSer -ScriptPath A32FF3AEAA23\reverse.bat
